# Experiment Guard — 實驗設計檢查模組

## 目的

在 AI 設計或執行實驗時，防止方法論錯誤、統計謬誤、和程式碼缺陷。
已知 AI 自動實驗有 42% 因 coding error 失敗——本模組的目標是在執行前攔截這些問題。

---

## 檢查量表

### 1. 研究設計適當性

```
□ 選用的研究方法是否適合回答提出的研究問題？
  - 探索性問題 → 質性或描述性研究
  - 相關性問題 → 觀察性研究 + 相關分析
  - 因果性問題 → 實驗設計（RCT 或準實驗）
□ 是否有明確的自變項和因變項？
□ 控制組的設計是否適當？
□ 是否說明了為什麼選這個方法而非其他替代方法？
```

### 2. 統計設計

```
□ 樣本量是否足夠？是否做了 power analysis？
  - 效果量（effect size）的假設是否合理且有文獻依據？
  - 顯著性水準（alpha）和統計效力（power）是否明確設定？
□ 使用的統計檢定是否適合資料類型？
  - 連續 vs. 類別、常態分佈 vs. 非常態
  - 獨立樣本 vs. 配對樣本
□ 多重比較是否做了校正（Bonferroni, FDR 等）？
□ 是否有預先定義的分析計畫（防止 p-hacking）？
□ 是否區分了主要分析和探索性分析？
```

### 3. 偏誤控制

```
□ 是否辨識了潛在的混淆變項（confounders）？
□ 是否有對應的控制策略？
  - 隨機分配、配對、統計控制、分層
□ 是否考慮了選擇偏誤（selection bias）？
□ 是否考慮了倖存者偏誤（survivorship bias）？
□ 對於 ML 實驗：是否有獨立的測試集，而非只報告訓練集結果？
□ 是否避免了資料洩漏（data leakage）？
```

### 4. 程式碼品質（針對計算實驗）

```
□ 程式碼是否能成功執行而不報錯？
□ 隨機種子是否已固定以確保可重現性？
□ 資料前處理步驟是否有文件記錄？
□ 是否有 sanity check（在小資料集上先跑一次確認邏輯正確）？
□ 輸出的數值是否在合理範圍內？
  - 準確率 > 99% → 需要懷疑是否有資料洩漏
  - Loss 為 NaN 或 Inf → 明確的程式問題
  - 結果在所有條件下完全相同 → 可能沒有真正跑不同條件
□ 是否有 error handling 防止靜默失敗？
```

### 5. 倫理考量

```
□ 是否涉及人類受試者資料？
  - 是 → 是否需要/已有 IRB 或倫理委員會核准？
  - 是 → 是否有知情同意程序？
  - 是 → 資料是否已適當去識別化？
□ 是否涉及敏感群體（兒童、病患、弱勢族群）？
□ 是否有潛在的雙重用途（dual-use）風險？
□ 是否考慮了公平性（不同群體的結果是否有差異）？
```

## 實驗設計反模式（紅旗清單）

以下是 AI 設計實驗時常見的反模式，出現任一項都應觸發警告：

| 反模式 | 說明 | 修正方向 |
|--------|------|---------|
| 目標指標不一致 | 實驗目標是 A，但報告 B 指標的改善 | 確認評估指標與研究問題對齊 |
| 隱性資料洩漏 | 測試集資訊滲入訓練過程 | 檢查資料分割時序和程式碼順序 |
| 超參數碰運氣 | 大量嘗試超參數但只報告最佳結果 | 報告所有嘗試或使用系統化的搜索策略 |
| 基線不公平 | baseline 使用較差的配置而非最佳實踐 | 用公認的 SOTA baseline |
| 只報告正面結果 | 省略失敗的實驗或負面結果 | 完整報告所有實驗結果 |
| Cherry-pick 指標 | 在多個指標中只挑有利的報告 | 預先定義主要指標 |

## 輸出格式

```
## 實驗設計檢查報告
- 研究設計：[類型] — 適當性 ✅/⚠️/❌
- 統計設計：樣本量 [N], 效果量 [d], 效力 [1-β]
- 偏誤風險：[辨識出的偏誤] + [控制策略]
- 程式碼檢查：[可執行 ✅/❌] [Sanity check ✅/❌]
- 倫理狀態：[N/A / 需要 IRB / 已有核准]
- 反模式掃描：[發現 N 個反模式] → [列出]
- 信心等級：🟢/🟡/🔴
```

## Study-Type Sub-modules

根據研究設計類型，載入對應的 sub-module：

- `experiment/rct.md` — 隨機化、盲性、ITT/PP、Cochrane RoB 2.0、CONSORT
- `experiment/observational.md` — 混淆控制、偏誤檢查、STROBE、因果語言校準
- `experiment/ml-experiment.md` — 資料分割/洩漏、baseline 公平性、超參數、反模式
- `experiment/qualitative.md`
- `experiment/math-symbol.md` — 符號定義、方程式一致性、維度檢查、符號表驗證
- `experiment/figures-tables.md` — 引用完整性、caption 品質、數據↔正文一致性、圖表品質信號 — 方法論一致性、飽和、編碼嚴謹性、可信度四標準
