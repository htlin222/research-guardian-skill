# Experiment Guard: Math & Symbol Consistency

## 何時載入
當研究包含方程式、數學符號、或形式化定義時。

---

## 數學一致性檢查

### 符號定義
```
□ 每個符號首次使用時是否有定義？
□ 同一符號在不同段落是否代表同一概念？
  - x 在 Methods 代表輸入，在 Results 代表不同的東西 → 🔴
□ 是否有未定義的符號？
□ 上下標是否一致？（x_i 和 x_j 的 i, j 代表什麼？）
□ 向量/矩陣是否有一致的標記法？（粗體、大寫、箭號）
```

### 方程式檢查
```
□ 方程式是否有編號？（被引用的方程式必須有）
□ 方程式是否在正文中被引用和解釋？
□ 方程式的維度是否一致？
  - 左邊是純量，右邊是向量 → 🔴
□ 方程式中的所有變數是否都已定義？
□ 求和/積分的範圍是否明確？
□ 是否有明顯的數學錯誤？
  - 分母可能為零
  - 概率超出 [0,1]
  - 負的距離/面積
```

### 符號表（如有）
```
□ 符號表是否與正文一致？
□ 正文中使用但符號表中缺失的符號？
□ 符號表中列出但正文中未使用的符號？
```
