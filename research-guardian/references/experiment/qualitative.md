# Experiment Guard: Qualitative Research

## 何時載入
當研究設計為質性研究（現象學、扎根理論、民族誌、個案研究、敘事研究等）時。

---

## 質性研究不適用量化 checklist

**重要：以下項目取代（不是補充）量化的 power analysis 和統計檢定要求。**
質性研究的嚴謹性由不同的標準衡量。

## 設計檢查清單

### 方法論一致性
```
□ 是否明確說明採用的質性傳統？
  - 現象學 (Phenomenology) → Moustakas, van Manen, Smith (IPA)
  - 扎根理論 (Grounded Theory) → Glaser & Strauss, Charmaz, Corbin
  - 民族誌 (Ethnography) → Geertz, Hammersley & Atkinson
  - 個案研究 (Case Study) → Yin, Stake, Merriam
  - 敘事研究 (Narrative) → Clandinin & Connelly, Riessman
□ 方法的選擇是否與研究問題匹配？
  - "What is the experience of X?" → 現象學
  - "How does process X work?" → 扎根理論
  - "What is the culture of X?" → 民族誌
□ 分析方法是否與傳統一致？
  - IPA 用 IPA 的分析步驟，不是 thematic analysis
```

### 抽樣
```
□ 抽樣策略是否適當且明確？
  - 目的性抽樣 (purposive) → 標準是什麼？
  - 理論性抽樣 (theoretical) → 扎根理論專用
  - 雪球抽樣 (snowball) → 是否報告了起始點？
□ 樣本量是否合理？
  IPA: 6-25（建議）
  Grounded Theory: 15-30（達到理論飽和）
  Ethnography: 取決於場域，通常 1-few 場域
  Case Study: 1-10 cases
  ⚠ 不要用量化的 power analysis 標準判斷質性研究的 N
□ 是否報告了飽和（saturation）？
  - 在哪一位參與者達到飽和？
  - 是否有額外的確認訪談？
□ 是否討論了樣本的多樣性/同質性？
```

### 資料收集
```
□ 訪談指引是否報告（或可取得）？
□ 訪談時長和方式（面對面/視訊/電話）？
□ 是否報告了資料記錄方式（錄音/錄影/筆記）？
□ 逐字稿是否由參與者確認（member checking）？
□ 研究者的角色和立場是否說明（reflexivity）？
```

### 分析嚴謹性
```
□ 編碼過程是否透明？
  - 至少兩位研究者獨立編碼？
  - 不一致如何解決？
  - 是否有編碼簿 (codebook)？
□ 主題的產生是否有證據支持？
  - 每個主題是否有多個參與者的引述支持？
  - 是否報告了反例 (deviant cases)？
□ 是否維持了 audit trail？
```

### 可信度 (Trustworthiness) — 質性研究的「效度」
```
Lincoln & Guba (1985) 四標準：
□ Credibility（可信性）→ 對應量化的 internal validity
  - Member checking? Peer debriefing? Prolonged engagement?
□ Transferability（可遷移性）→ 對應 external validity
  - Thick description? 讓讀者判斷是否適用於自己的情境？
□ Dependability（可靠性）→ 對應 reliability
  - Audit trail? 外部審核？
□ Confirmability（可確認性）→ 對應 objectivity
  - Reflexive journal? 三角驗證 (triangulation)?
```

### 倫理
```
□ 知情同意？
□ 匿名/化名處理？
□ 特別是脆弱群體的額外保護？
□ 研究者與參與者的關係（power dynamics）是否討論？
```

```
- 抽樣：[策略] N=[N] — 飽和 [已達/未報告]
- 分析嚴謹性：獨立編碼 [✅/❌] audit trail [✅/❌]
- 可信度：credibility [✅/❌] transferability [✅/❌]
             dependability [✅/❌] confirmability [✅/❌]
- 信心等級：🟢/🟡/🔴
```
