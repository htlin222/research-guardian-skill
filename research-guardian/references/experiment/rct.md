# Experiment Guard: Randomized Controlled Trial

## 何時載入
當實驗設計為隨機對照試驗時。

---

## RCT 設計檢查清單

### 隨機化
```
□ 隨機化方法是否適當？
  - 簡單隨機 / 區塊隨機 / 分層隨機
  - 偽隨機（按入院順序、生日奇偶）→ ❌ 不接受
□ 隨機序列產生方法是否報告？
□ 分配隱匿（allocation concealment）是否做到？
  - 中央隨機化 / 密封信封（不透光）
□ 隨機化後兩組的 baseline 是否平衡？
  - 如果不平衡 → 是否做了調整分析？
```

### 盲性
```
□ 盲性等級？
  - 開放標籤（open-label）→ 偏誤風險高
  - 單盲（受試者不知）→ 中等
  - 雙盲（受試者+研究者不知）→ 最佳
  - 三盲（+分析者不知）→ 理想
□ 盲性是否被破壞？如何評估？
□ 安慰劑是否匹配？（外觀、口味、給藥方式）
```

### 樣本量
```
□ 是否做了事前 power analysis？
  - α = 0.05, power = 0.80 (最低), 0.90 (建議)
  - 效果量假設基於什麼？（先導研究？文獻？）
□ 退出/流失率是否可接受？
  - > 20% 流失 → 結果可信度下降 ⚠
□ ITT (Intention-to-Treat) vs. PP (Per-Protocol) 分析？
  - ITT = 保守但公平, PP = 可能高估效果
  - 應兩者都報告
```

### 結局指標
```
□ 主要結局（primary outcome）是否預先定義？
□ 主要結局是否為臨床相關（非替代指標）？
□ 次要結局數量是否合理？（太多 → 多重比較問題）
□ 結局評估時間點是否合理？
□ 結局評估者是否盲性？
```

### 偏誤風險（Cochrane RoB 2.0）
```
評估 5 個 domain：
□ D1: 隨機化過程中的偏誤
□ D2: 偏離預定介入的偏誤
□ D3: 結局數據缺失的偏誤
□ D4: 結局測量的偏誤
□ D5: 選擇性報告結果的偏誤
每個 domain：Low / Some concerns / High risk
```

### 報告標準
```
□ 是否遵循 CONSORT 聲明？
□ CONSORT 流程圖是否完整？
□ 試驗是否預先註冊？（ClinicalTrials.gov / ISRCTN）
□ 是否報告了預註冊與實際執行的偏差？
```

## 輸出格式
```
## RCT 設計檢查補充
- 隨機化品質：[適當/可疑/不當]
- 盲性等級：[open/single/double/triple]
- Power：α=[X] β=[X] 效果量=[X] N=[X]
- RoB 評估：D1=[L/S/H] D2=[L/S/H] D3=[L/S/H] D4=[L/S/H] D5=[L/S/H]
- CONSORT 完整性：[完整/部分/缺失]
```
