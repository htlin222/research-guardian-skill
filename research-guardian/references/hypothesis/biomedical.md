# Hypothesis Guard: Biomedical Domain

## 何時載入
當研究假設涉及醫學、藥學、公共衛生、臨床研究、基因體學、或任何涉及人類健康的主題時。

---

## Biomedical 假設的特殊要求

### 1. 臨床可轉譯性檢查
```
□ 假設是否有明確的臨床意義？（不只是統計意義）
□ 如果假設成立，對臨床實踐的改變是什麼？
□ 目標族群是否明確定義？（年齡、性別、疾病分期、共病）
□ 預期的效果量是否達到臨床上有意義的門檻？
  - 最小臨床重要差異 (MCID) 是多少？
  - NNT (Number Needed to Treat) 是否合理？
```

### 2. 生物學合理性
```
□ 假設是否有已知的生物學機制支撐？
□ 是否有動物模型或體外實驗的前期證據？
□ 假設的因果路徑是否合理？
  - 暴露 → 生物學機制 → 結果
  - 每一步是否都有文獻支持？
□ 是否與已知的病理生理學一致？
  - 如果不一致，是否有充分理由挑戰現有知識？
```

### 3. 現有證據等級評估
```
既有證據           → 假設可以做的宣稱
────────────────────────────────────────
無前期數據         → "探索性假設"
體外/動物實驗      → "值得在人體中驗證"
觀察性研究         → "可能存在關聯"
小規模 RCT         → "初步證據支持"
多項 RCT / SR      → "在此基礎上進一步探索"
```

### 4. 安全性與倫理紅線
```
□ 假設是否涉及已知有害的介入？
□ 是否有 equipoise（真正的不確定性）支持進行 RCT？
  - 如果已有明確證據表示某治療有效，再做安慰劑對照是不倫理的
□ 弱勢族群（兒童、孕婦、認知障礙者）是否需要特別保護？
□ 是否有利益衝突需要揭露？
```

### 5. Biomedical 文獻搜索特殊策略
```
必查資料庫：
- PubMed / MEDLINE（首選）
- Cochrane Library（系統性回顧）
- ClinicalTrials.gov（進行中的試驗）
- WHO ICTRP（國際試驗登記）

必查的搜索面向：
- [疾病/條件] AND [介入/暴露]
- [介入] AND "systematic review"
- [介入] AND "randomized controlled trial"
- [介入] AND "adverse effects" OR "safety"

特殊注意：
- 藥物假設必須檢查 FDA/EMA 核准狀態
- 基因治療假設必須檢查 CRISPR 相關倫理指引
- 再利用藥物假設必須確認原始適應症和安全性資料
```

### 6. Biomedical 常見假設陷阱
```
⚠ 替代終點陷阱：用替代指標代替臨床結果
  例：「降低血壓 → 減少心血管事件」不一定成立
  修正：明確區分替代終點和臨床終點

⚠ 健康使用者偏誤：接受預防措施的人本來就比較健康
  例：「補充維他命的人更長壽」可能是混淆
  修正：討論 healthy user bias 的可能性

⚠ 時間窗口偏誤：治療效果取決於介入時機
  例：同一藥物在疾病早期和晚期效果可能相反
  修正：假設中明確時間窗口

⚠ 劑量-反應假設：不能假設線性關係
  例：「越多越好」在藥理學中經常不成立（U 型曲線）
  修正：假設中包含劑量範圍和非線性可能
```

## ML + 臨床決策的額外要求
```
如果研究涉及 ML 模型用於臨床決策支援：
□ 是否有 interpretability / explainability 分析？
  - Grad-CAM, SHAP, LIME, attention visualization
  - 臨床醫師能否理解模型的決策依據？
□ 是否有 failure mode analysis？
  - 模型在哪些子群體上表現差？
  - 錯誤案例是否有臨床可解釋的模式？
□ 是否討論了 AI 決策和臨床判斷不一致時的處理方式？
```

## 輸出格式

```
## Biomedical 假設驗證補充報告
- 臨床可轉譯性：[HIGH/MEDIUM/LOW] — [MCID 評估]
- 生物學合理性：[支持機制] / [反面證據]
- 現有證據等級：[等級] — [關鍵文獻]
- 安全性初評：[已知風險] / [未知風險]
- Equipoise 狀態：[存在/不存在/不確定]
```
