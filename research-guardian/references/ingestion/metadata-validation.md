# Metadata Validation — 後設資料驗證

## 何時載入
Paper Parser 完成後，在 gate 派發前。

---

## 驗證項目

### 作者與機構
```
□ 作者姓名格式是否一致？（全名 vs. 縮寫）
□ 所有作者是否都有機構歸屬？
□ 通訊作者是否標明？
□ 作者貢獻聲明（CRediT）是否存在？
□ ORCID 是否提供？（建議但非必須）
```

### 發表資訊
```
□ 標題是否完整且無截斷？
□ 目標期刊/會議是否明確？
□ DOI 是否存在且可解析？
□ 投稿日期/接受日期是否合理？
□ 版本號（如 arXiv）是否為最新？
```

### 利益衝突
```
□ 利益衝突聲明是否存在？
□ 資金來源是否揭露？
□ 資金來源是否可能影響結論？（如藥廠贊助的藥物研究）
  → 不是自動 FAIL，但需標記為 context
```

### 資料與程式碼可用性
```
□ 是否有 data availability statement？
□ 如果宣稱 open data → 連結是否有效？
□ 如果宣稱 open code → repo 是否存在且非空？
□ 如果資料不公開 → 是否說明原因？
```
