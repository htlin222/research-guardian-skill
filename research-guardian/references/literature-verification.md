# Literature Guard — 文獻驗證模組

## 目的

防止 AI 在引用文獻時最常見的致命錯誤：
- 引用不存在的論文（幻覺引用）
- 錯誤歸因（把 A 說的話歸給 B）
- 斷章取義（引用內容不反映原文意思）
- 遺漏關鍵文獻（該引的沒引）
- 引用過度老舊

**幻覺引用是 AI 研究中最容易被抓到、也最致命的錯誤。一篇假引用就足以摧毀整份研究的可信度。**

## 檢查量表

### 引用驗證分級

根據引用的重要性分級驗證：

```
Tier 1 — 關鍵引用（研究設計的基礎、核心方法的來源）
  → 完整三重驗證 + retraction check
  → evaluator 必須 fetch 原文摘要，逐句確認引用描述是否準確
  → 任何問題 = 整個研究信心降級

Tier 2 — 支持引用（支持論點但非基礎）
  → 至少完成標題搜索驗證

Tier 3 — 背景引用（一般背景知識）
  → 存在性確認即可

判定方法：如果移除該引用，核心論點是否站不住？是=Tier 1
```

### SR 摘要引用標準

SR（Systematic Review）摘要不需要列出所有納入研究的引用。
SR 正文必須完整列出。SR 摘要的引用覆蓋度為 advisory，不觸發 FAIL。

### 模糊引用判定規則

```
有標題+作者+年份+期刊    → 可驗證，進入三重驗證
有標題+作者+年份（無期刊）→ 部分可驗證，標記 🟡
只有作者+年份（無標題）   → 不可驗證 → 🔴 FAIL
只有 "studies show"       → 🔴 FAIL（懸空引用）

原則：舉證責任在引用者。
```

### 1. 引用存在性驗證（必做，零容忍）

```
□ 每一篇引用的論文，是否都能透過以下至少一種方式確認存在？
  - Google Scholar 搜索 → 找到完全匹配的標題、作者、年份
  - DOI 連結 → 可以正常解析到論文頁面
  - Semantic Scholar / PubMed 查詢 → 有對應記錄
□ 如果無法確認存在，是否已移除該引用？
□ 作者姓名拼寫是否正確？
□ 年份是否正確？
□ 發表期刊/會議名稱是否正確？
```

**操作方式（三重驗證）：**

1. **第一道：標題精確搜索**
   - 取引用的完整標題，加引號搜索
   - 如果找不到完全匹配 → 標記為可疑

2. **第二道：作者 + 年份 + 關鍵詞搜索**
   - 用第一作者姓氏 + 年份 + 2-3 個主題關鍵詞
   - 可能找到實際存在但標題被 AI 稍微改動的論文

3. **第三道：DOI 或 URL 驗證**
   - 如果有 DOI，直接解析 doi.org/xxx
   - 如果有 URL，嘗試 fetch 確認頁面存在

**紅線規則：**
- 三道都無法確認 → 必須移除該引用，不能保留
- 部分匹配（標題相似但不完全一致）→ 標記為 🟡 並附註差異
- AI 不得「修正」一篇找不到的引用使其看起來正確——這比不引用更危險

### 2. 引用準確性

```
□ 引用的觀點是否正確反映原文的主要論述？
□ 是否有斷章取義的風險（原文的結論可能更 nuanced）？
□ 引用的數據（數字、百分比）是否與原文一致？
□ 是否區分了原文的「發現」和「推測」？
```

**操作方式：**
- 對關鍵引用（支撐核心論點的引用），嘗試獲取原文摘要
- 比對 AI 的引用描述與原文摘要是否一致
- 特別注意量化數據——數字最容易被幻覺

### 3. 覆蓋度檢查

```
□ 是否涵蓋了該領域最重要的 3-5 篇 foundational papers？
□ 是否包含了最近 2 年內的相關文獻？
□ 是否包含了反面觀點或批評性文獻？
□ 是否跨越了相鄰領域的重要文獻（避免 silo effect）？
□ 引用的來源是否多元（不只集中在 1-2 個研究團隊）？
```

**操作方式：**
- 搜索 "[主題] survey" 或 "[主題] review" 找到已有的綜述文章
- 從綜述文章的引用列表中檢查是否遺漏了高引用的關鍵文獻
- 搜索 "[主題] criticism" 或 "[主題] limitation" 確保包含反面聲音

### 4. 新近性檢查

```
□ 引用文獻的中位年份是多少？是否在 5 年內？
□ 是否有超過 10 年且非 foundational 的引用需要更新？
□ 是否確認了引用的資訊沒有被後續研究推翻？
```

**已知失敗案例：**
AI Scientist v1 的論文中位引用數僅 5 篇，34 篇引用中只有 5 篇是 2020 年之後的。

## 文獻搜索策略指引

### 深度搜索 vs. 廣度搜索

| 場景 | 策略 | 搜索次數建議 |
|------|------|------------|
| 驗證單一引用 | 精確搜索 | 1-3 次 |
| 文獻回顧 | 先廣後深 | 5-10 次 |
| Novelty 確認 | 多角度搜索 | 5-8 次（不同關鍵詞組合）|
| Systematic review | 系統性搜索 | 15-20 次 + 滾雪球 |

### 搜索關鍵詞設計

- **正向搜索**：[核心概念] + [方法] + [領域]
- **反向搜索**：[核心概念] + "limitation" / "failure" / "criticism"
- **相鄰搜索**：[相關但不同的術語] + [同一問題]
- **時間限制搜索**：加入年份範圍確保獲取最新文獻

### 滾雪球法

當找到一篇高度相關的文獻時：
1. 檢查其引用列表（向後追溯）
2. 搜索「誰引用了這篇」（向前追蹤）
3. 檢查同一作者的其他相關著作

## Domain-Specific Sub-modules

根據文獻所屬領域，載入對應的 sub-module：

- `literature/biomedical.md` — 證據等級標記、指引版本、retraction 檢查、SR 引用規範
- `literature/cs-ml.md` — 預印本 vs. 同行審查、版本問題、SOTA 時效、產業 vs. 學術
- `literature/social-science.md`
- `literature/gap-framing.md` — Gap 真實性、先前工作描述公平性、Gap↔Contribution 對齊 — 再現性狀態標記、效果量校準、量表效度、WEIRD 標記
