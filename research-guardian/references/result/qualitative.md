# Result Guard: Qualitative Research Results

## 何時載入
當結果報告為質性研究的主題/類別/理論時。

---

## 質性結果驗證（取代量化的統計檢查）

### 主題/類別的證據支持度
```
□ 每個主題是否有至少 2-3 位不同參與者的引述支持？
  - 單一參與者的引述不足以支撐一個主題
□ 引述是否真的支持該主題？（不是斷章取義）
□ 主題之間是否有清楚的區分？（不重疊、不冗餘）
□ 主題的命名是否反映了引述的內容？
  - 過度抽象的主題名 → 可能脫離數據
  - 過度具體的主題名 → 可能缺乏分析深度
```

### 反例處理 (Deviant Case Analysis)
```
□ 是否報告了與主要主題不一致的案例？
  - 全部一致 → 可疑（真實數據幾乎總有變異）
  - 報告了不一致但解釋了原因 → ✅
  - 完全未提反例 → 🟡 WARN
□ 反例是否改變了主題的結構或限定了適用範圍？
```

### 引述的充分性和倫理
```
□ 引述是否匿名化？
□ 引述的長度是否足以理解脈絡？（不是只有一句話片段）
□ 引述的分佈是否均衡？（不是只引用 2-3 位最能說的參與者）
□ 是否報告了沉默或拒絕回答的情況？
```

### 結論適度性（質性版本）
```
允許的語言：
  ✅ "Participants described..." / "Themes suggest..."
  ✅ "In this context, X appeared to..."
  ✅ "These findings indicate that for [specific group]..."
  
不允許的語言：
  ❌ "This proves that all [group] experience..."
  ❌ "The definitive pattern is..."
  ❌ 從質性數據直接推論因果
  
□ 結論是否限定在研究的特定脈絡中？
□ 是否避免了從少數參與者推論到整個群體？
□ 是否使用了 transferability（而非 generalizability）的概念？
```

### 理論建構（扎根理論專用）
```
□ 核心類別 (core category) 是否清楚？
□ 理論是否「扎根」在數據中（而非先驗假設）？
□ 類別之間的關係是否有清楚的論述？
□ 理論飽和是否達到？
```

---

## 質性結果的信心等級標準
```
🟢 HIGH:  所有主題有多重引述支持 + 報告反例 + 語氣適度
🟡 MEDIUM: 主題有支持但部分薄弱 + 反例未充分討論
🔴 LOW:  主題缺乏引述支持 + 無反例 + overclaim
```

## 輸出格式
```
## 質性結果驗證
- 主題數量：[N]
- 引述支持度：每個主題平均 [N] 個引述
- 引述分佈：[均衡/集中在少數參與者]
- 反例報告：[有/無]
- 結論適度性：[適度/過度推論]
- 信心等級：🟢/🟡/🔴
```
