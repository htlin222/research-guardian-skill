# Writing Guard: Journal Paper

## 何時載入
當撰寫或驗證投稿至同行審查期刊的論文時。

---

## 期刊論文結構檢查
```
□ Title: 是否精確反映內容？避免過度承諾？
□ Abstract: 是否獨立可讀？包含目的、方法、結果、結論？
  - 字數是否符合目標期刊要求？（通常 150-300 字）
□ Introduction: 是否建立了 research gap？
  - 現有知識 → 不足之處 → 本研究的定位
□ Methods: 是否足夠詳細到他人可重現？
□ Results: 是否只報告發現，不含解讀？
□ Discussion: 是否回應了 Introduction 的問題？
  - 主要發現的意義
  - 與現有文獻的比較
  - 替代解釋
  - 局限性（至少 3 項具體的）
  - 未來方向
□ Conclusion: 是否為 2-3 句精簡總結？不重複 Discussion？
```

## 期刊特定要求
```
□ 目標期刊的 author guidelines 是否已檢查？
□ 格式要求：APA / AMA / Vancouver / IEEE / 自訂？
□ 字數限制：正文、摘要、參考文獻數量？
□ 圖表數量限制？
□ supplementary materials 的規範？
□ 利益衝突聲明？
□ 資料共享政策？
□ 倫理聲明（human subjects, animal research）？
□ 作者貢獻聲明（CRediT）？
```

## 審稿人心理模型
```
審稿人最常見的拒稿理由（按頻率排序）：
1. Novelty 不足 → 確認 contribution 在 Introduction 中清楚標示
2. 方法論問題 → 確認 Methods 足夠嚴謹
3. 結論過度推論 → 確認 Discussion 中有適當 hedging
4. 文獻回顧不完整 → 確認覆蓋了關鍵文獻
5. 寫作品質差 → 確認語法和邏輯流暢

期刊 vs. 會議差異：
- 期刊更重視完整性和嚴謹性
- 期刊允許更長的 Methods 和 Discussion
- 期刊期望更完整的文獻回顧
- 修改機會：Major/Minor revision 是常見的（不是拒稿）
```

## 語氣規範
```
期刊論文的正式度最高：
- 避免第一人稱（某些領域允許 "we"）
- 避免口語化表達
- 避免過度 hedging（"it might perhaps be possible that..."）
- 使用被動語態適度（不要全篇被動）
- 數字 < 10 用文字拼寫（field-specific）
```

## 輸出格式
```
## Journal 寫作驗證補充
- 結構完整性：[所有必要章節 ✅ / 缺少 [章節]]
- 格式合規：[符合目標期刊 / 需調整 [項目]]
- 審稿人風險：[最大風險：[issue]]
```
